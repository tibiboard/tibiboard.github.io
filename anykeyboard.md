# 他のZMKキーボードをちびボードアプリ対応にする

[ちびボードの設定アプリ(ブラウザ版)](https://tibiboard.github.io/app/)は、
ZMK Studio対応のキーボードなら**どれでもキーマップ編集**に使えます。

さらに、ファームウェアを「ちびボード版ZMK」でビルドし直すと、
**コンボ・タップダンス・マクロの編集**もフルに使えるようになります。
このページはそのやり方です(所要30分くらい・はんだ付け等は不要)。

## 必要なもの

- ZMK対応の自作キーボード(ZMKで動いていれば機種は問いません)
- GitHubアカウント(無料。作り方は検索するとたくさん出てきます)

## 手順(3ステップ)

### ① テンプレートから自分のリポジトリを作る

1. [tibiboard/tibiboard30-](https://github.com/tibiboard/tibiboard30-) を開く
2. 緑の「**Use this template**」→「Create a new repository」
3. **必ず「Public」を選ぶ**(Privateだと無料プランでは自動ビルドが動きません)

### ② 自分のキーボードに差し替える

作ったリポジトリで、次の2箇所を編集します(GitHubの画面上で鉛筆アイコンから編集できます):

1. **build.yaml** — シールド名を自分のキーボードのものに変える
   ```yaml
   include:
     - board: nice_nano_v2        # 自分のキーボードのボード名
       shield: corne_left         # 自分のキーボードのシールド名
       snippet: studio-rpc-usb-uart   # ←アプリと話す側(左/central)に必ず付ける
     - board: nice_nano_v2
       shield: corne_right
   ```
   ZMK本家に入っている機種(Corne, Sofle, Lily58など)はシールド名を書くだけでOK。
   自作シールドの場合は `config/boards/shields/` に自分のシールド一式を置いてください。

2. **自分のキーマップ(config/◯◯.keymap)** — 先頭付近に1行追加:
   ```c
   #include "spare_slots.dtsi"
   ```
   `spare_slots.dtsi` はテンプレートに同梱されています(アプリが書き換えるための
   「空のコンボ/タップダンス/マクロ枠」の定義です)。

### ③ ビルドしてダウンロード

1. リポジトリの「**Actions**」タブ → 「Build」→「Run workflow」
2. 数分待つと、ページ下部の **Artifacts** に `.uf2` ファイルができます
3. ダウンロードして、いつもの方法でキーボードに書き込めば完成

[ブラウザ版アプリ](https://tibiboard.github.io/app/)にUSBでつなぐと、
コンボなどのタブが自動で現れます。

## よくある質問

- **振動タブが出てきた/鳴らない** — 振動モーターが無い機種では押しても何も起きないだけです(害はありません)
- **キーマップ画面の見た目** — あなたのキーボードの形がそのまま表示されます(ZMK Studioの標準機能)
- **元のファームに戻したい** — 元のuf2を書き込み直すだけです。いつでも戻れます
- **うまくいかない** — ビルドが赤くなったらActionsのログを見てください。よくある原因は「Privateで作ってしまった」「シールド名の綴り」です

## 動作確認チェックリスト

- [ ] アプリに接続してコンボのタブが表示される
- [ ] コンボを1個作る → 実機で発動する → 削除できる
- [ ] 電源を入れ直しても設定が残っている
- [ ] 既存のキーの動きに影響がない
